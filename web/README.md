# foodfight (web版)

`processing/` にあった旧アプリを、Scratch/Chrome拡張を使わずブラウザから直接toio 2台とBluetooth(Web Bluetooth API)で通信するように書き直したもの。ゲームロジック(当たり判定・パーティクル演出・射影変換キャリブレーション・サウンド)は元のProcessingコードとほぼ1:1で移植している。

## 起動方法

Web Bluetoothは`file://`では動かないため、必ず簡易HTTPサーバー経由で開く。

```sh
cd web
python3 serve.py
```

(`python3 -m http.server 8000` でも動くが、キャッシュ無効化ヘッダーを送らないため、コード更新後にブラウザが古いJS/CSSをキャッシュして反映されないことがある。開発中・会場での更新時は `serve.py` を推奨。それでも反映されない場合はブラウザで `Cmd+Shift+R` のハードリロードを試すこと。)

Chromeで `http://localhost:8000` を開く(Safari/Firefoxは Web Bluetooth 非対応/一部対応のため非推奨)。

**毎回同じオリジン(同じホスト・ポート)を使うこと。** ChromeのBluetooth接続許可はオリジン単位のため、ポートを変えると再ペアリングが必要になり、自動再接続(`getDevices()`)も効かなくなる。

## 実機なしでの動作確認(モックモード)

`http://localhost:8000/?mock=1` でtoio実機なしにゲームロジックを確認できる。
- Cube1: W/A/S/D
- Cube2: 矢印キー

## 会場セットアップ手順

1. プロジェクタを拡張ディスプレイとして接続した状態で、画面左上の「フルスクリーン(プロジェクタへ)」ボタンを押す。Window Management API(Chrome)が使える環境では、初回のみ画面選択の許可ダイアログが出た後、自動的にプライマリでない方のディスプレイ(=プロジェクタ)にフルスクリーン表示される。ウィンドウを手動でドラッグする必要はない。
   - 許可ダイアログを拒否した場合や非対応ブラウザの場合は、従来通り「ブラウザウィンドウを手動でプロジェクタ側にドラッグしてからボタンを押す」動作にフォールバックする。
   - それでも正しいディスプレイに出ない場合は、macOSの設定を確認: システム設定→ディスプレイで「ミラーリング」ではなく「拡張」になっているか、システム設定→デスクトップとDockの「ディスプレイごとに個別のSpaceを使用」が有効になっているか。
2. 「Connect Cube 1」「Connect Cube 2」を押し、それぞれtoioキューブをペアリングする(ネイティブのデバイス選択ダイアログが1台ずつ表示される)。キューブ本体に `toio-XXX` のBLE名シールを貼っておくと選び間違いを防げる。
   - 画面左下に生のx/y/angleと最終更新からの経過時間を表示するデバッグパネルが常時出ている。キューブを動かして数値が追従すればBLE通信は正常。
3. `c` キーでキャリブレーションモードに入り、プロジェクタ画面の4隅を左上から時計回りにクリックする(`js/constants.js` の `MAT_SOURCE_CORNERS` には、公式の toio™開発用プレイマット TMD01SS(#01-#06簡易プレイマット面)のPosition ID座標の公称値を直接ハードコードしてあるため、マット実測は不要)。
4. 黒い背景をクリックして食べ物を配置し、`s` キーで保存する。
5. `1` キーで準備モードに入る。両キューブをスタート位置(マット中央からやや左右、`js/constants.js`の`PLAYER1_START`/`PLAYER2_START`付近)に置くと自動的にゲームが始まる。`2` キーでも(デバッグ用に)即座にゲーム開始できる。

**別のマットを使う場合の注意:** `MAT_SOURCE_CORNERS`はTMD01SSの#01-#06面専用の値。裏面(連番マット)やtoioコレクションのプレイマットなど別のシートを使う場合は値が異なるため、左下のデバッグパネルでキューブをマットの4隅に置いた際の生のx/y値を読み取り、`MAT_SOURCE_CORNERS`を実測値に置き換えること。

## データのバックアップ

キャリブレーション(`warp.json`相当)・食べ物配置(`food.json`相当)は`localStorage`に保存される。ブラウザデータが消えると失われるため、**本番のキャリブレーション後は必ず「エクスポート」ボタンでJSONファイルとして保存しておくこと**。復元は「warp.json 読込」「food.json 読込」ボタンから行う。

## 既存フォルダとの関係

`processing/`, `chrome拡張/`, `toio_scratch/` は変更せず残してある(過去バージョンの参照用)。この`web/`はそれらに依存しない独立した実装。

## 既知の制約

- Web Bluetoothの`requestDevice()`はユーザー操作からのみ呼べるため、Cube1/Cube2それぞれ個別のボタンクリックでペアリングする(1回のダイアログで2台同時選択はできない)。
- ブラウザの自動再生ポリシーにより、最初のボタンクリックまで音声が鳴らない場合がある(Connect/フルスクリーンボタンのクリックで解除される設計にしてある)。
- 「フルスクリーン(プロジェクタへ)」はWindow Management API(Chrome)を使ってプライマリでないディスプレイへ自動的にフルスクリーン表示する。非対応ブラウザ/権限拒否時は手動でウィンドウをプロジェクタ側にドラッグしてからボタンを押す動作にフォールバックする。
